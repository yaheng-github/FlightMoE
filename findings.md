# FlightMoE 研究发现

## 数据集洞察

### RflyMAD 数据结构
- 41 维传感器字段，包含 IMU、PWM、电池、电机、控制量等
- 三种数据模态：原始时序、时频图像 (STFT)、飞控日志
- 时间戳精度：10ms (对多模态对齐至关重要)
- 真实飞行样本仅 497 个，sim-to-real 泛化是关键挑战

### 故障类型分布特点
- 电机故障样本最多 (SIL/HIL 各 921)，是主要检测目标
- 风场扰动是完美的"负样本"：传感器波动大但物理一致性未破坏
- 真实飞行中缺少螺旋桨故障、低电压、风场扰动、载荷丢失数据

## 技术方案洞察

### 阶段嵌入的重要性
- 不同飞行阶段正常基线差异巨大：起飞时高动态波动是正常的，悬停时同样幅度就是异常
- 阶段 Token 应与传感器特征深度语义融合，而非简单拼接

### 物理一致性图的构建策略
- 可用 FP-Growth 从正常数据自动挖掘强耦合关系
- 可借鉴 RflyMAD 论文中 EKF 预测方程建立"位置↔速度↔姿态"边约束
- 可根据无人机物理结构 (电机力臂、质量、转动惯量) 赋予几何初始权重

### 路由网络设计要点
- 风场扰动场景：传感器数值剧烈震荡但物理关联规则依然成立 → 路由应降低异常专家置信度
- 模态缺失场景：GPS 丢星时，Router 应自动屏蔽 Burst Expert (依赖位置突变)，提升 Consistency Expert 权重

### 反事实训练关键
- 单纯噪声/同步误差 → 要求输出与干净样本一致 (L_inv)
- 违反物理规律的扰动 → 强制 Consistency Expert 报极高残差 (L_consist)
- 模态缺失 → 强制 Router 重组专家权重 (L_route)

## 基线方法选择依据

| FlightMoE 模块 | 基线论文 | 选择理由 |
|---------------|---------|---------|
| Burst Expert | MAD-GAN | GAN 学习多维联合分布，DR-Score 双指标，适合突变检测 |
| Drift Expert | HMM + 对抗增强 | 对时间序列状态转移建模，适合渐发故障 |
| Spectral Expert | GANomaly | Encoder-Decoder-Encoder 架构，潜在空间一致性损失，适合频域异常 |
| Consistency Expert | GDN | GAT 架构 + 传感器嵌入 + 偏差得分量化，直接可用作代码底座 |
| Routing Network | Sparsely-Gated MoE | 稀疏门控 + 负载均衡，路由器设计图 |

## 潜在风险点

1. **41 维字段的归一化策略**: 不同传感器量纲差异巨大 (PWM 0-1000 vs 加速度 m/s²)，需要分组归一化
2. **STFT 参数选择**: 窗口大小和重叠率影响时频分辨率，需根据异常时间尺度调优
3. **GNN 的 scalability**: 41 个节点规模较小，但需确保实时推理速度满足 100Hz+ 要求
4. **Top-k 路由的 k 值**: k=1 过于激进，k=2 或 3 可能更稳健，需实验确定
5. **复合损失的权重平衡**: L_inv、L_consist、L_route 的相对权重对收敛影响大

## 灵感记录

- **可解释性增强**: 系统输出不仅给异常分数，还可给出"专家归因路径"——哪个专家发现了什么类型的异常
- **在线学习**: 真实飞行中持续收集正常数据，定期更新 FP-Growth 关联规则
- **边缘部署**: 稀疏路由天然适合资源受限环境，只有 2/4 专家激活时计算量减半
