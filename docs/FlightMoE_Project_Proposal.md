## 一、项目概述

### 1.1 项目名称

**FlightMoE**: Phase-Aware and Reliability-Guided Multimodal Expert Routing for Open-Set UAV Anomaly Detection

### 1.2 研究背景

随着低空经济快速发展，无人机已广泛应用于物流配送、巡检监测和农业植保等场景。然而，硬件故障、状态估计异常、传感器漂移和通信扰动仍频繁威胁飞行安全。

与传统工业异常检测不同，无人机异常具有显著的特征：

- **阶段依赖性**: 同一故障在不同飞行阶段（起飞、悬停、巡航、机动、降落）呈现不同动态形态
- **跨模态耦合性**: 不同模态对异常的响应强度和时延存在差异
- **物理一致性约束**: 噪声、丢包和时间错位会导致模态间冲突

现有方法多基于单模态时序建模或简单后融合，难以同时处理突变异常、漂移异常、频谱结构异常及模态可靠性差异。尤其是在模态缺失、模态冲突和 sim-to-real 偏移条件下，现有方法往往缺乏稳定的异常证据聚合机制。

### 1.3 核心目标

提出一个能够在不完整多模态观测下统一建模飞行阶段、异常形态和跨模态一致性的鲁棒异常检测框架，投稿至 **AAAI**。

## 二、现有技术问题分析

### 2.1 模型驱动方法

- 精确建模难度高，维护成本大
- 对未建模或弱先验故障模式适应性不足
- 多源数据利用不足，难以建立跨模态因果链路
- 对复杂环境与工况变化的鲁棒性有限

### 2.2 单模态数据驱动方法

- 局限于单一或同质数据模态，信息利用不充分
- 缺乏对异构数据原生结构的建模与协同分析能力
- 对未知或复合型异常的泛化能力存在瓶颈

### 2.3 多传感器/弱多模态方法

- 多为"多通道同构数据"，尚未覆盖日志、图像等强异构模态
- 缺乏对多模态数据的精细时空对齐与统一表示机制
- 缺少在融合空间中进行无监督学习与对抗增强的系统化设计

## 三、技术方案: FlightMoE 框架

FlightMoE 是一个面向无人机异常检测的**阶段感知、可靠性引导、物理一致性约束的多模态稀疏专家路由框架**。

### 3.1 系统总体架构

系统采用层次化的多智能体协同架构，分为五个功能层级：

```text
┌─────────────────────────────────────────────────────────────┐
│ 第五层: 决策输出与安全响应层                                    │
│  → 聚合多专家结论，输出故障归因图与异常等级，联动安全控制接口        │
├─────────────────────────────────────────────────────────────┤
│ 第四层: 稀疏专家路由与推理层                                    │
│  → 根据飞行阶段、信号稳定性及模态完整度，动态激活异常专家            │
├─────────────────────────────────────────────────────────────┤
│ 第三层: 物理一致性分析层                                       │
│  → 构建动态关联图，利用图学习进行物理校准，生成一致性残差            │
├─────────────────────────────────────────────────────────────┤
│ 第二层: 特征表示与对齐层                                       │
│  → 滑动窗口时间切片，异构编码器映射至统一高维特征空间               │
├─────────────────────────────────────────────────────────────┤
│ 第一层: 多模态数据输入层                                       │
│  → 实时同步采集执行器指令、动力学状态、环境感知及机载系统参数        │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 模块一: 阶段感知多视图编码器

**目标**: 统一建模原始时序、状态上下文和时频结构，覆盖突变、漂移和频谱型异常。

**输入视图**:
- **原始时序视图**: IMU、PWM、电池、电机、控制量等（41维传感器字段）
- **时频视图**: 对高频动态通道做 STFT，得到 spectrogram patch
- **阶段 Token**: 显式表示起飞、悬停、巡航、转向、降落等飞行阶段

**阶段嵌入机制**:
- 将实时飞行工况转化为特定标识向量
- 与传感器特征向量进行深度语义融合
- 编码层内部采用交互注意力机制，实现跨模态信息互补增强

**时频图像通道映射** (RGB 伪彩色编码):

| 图像组名 | R 通道 | G 通道 | B 通道 | 检测目标 |
|---------|--------|--------|--------|---------|
| Magnetometer | X轴地磁场强度 | Y轴地磁场强度 | Z轴地磁场强度 | 环境地磁场变化 |
| Gyro | X轴角速度 | Y轴角速度 | Z轴角速度 | 飞行姿态震荡、平衡环异常 |
| Accel | X轴加速度 | Y轴加速度 | Z轴加速度 | 结构松动、物理撞击、剧烈抖动 |
| Control | Roll 控制指令 | Pitch 控制指令 | Yaw 控制指令 | 飞控算法异常、控制链路干扰 |

### 3.3 模块二: 物理一致性驱动的跨模态关系建模

**目标**: 识别跨模态物理关系被破坏的异常，即使单模态本身看起来正常。

**核心思想**:
- 建立 physics-aware cross-modal consistency graph
- 正常时传感器间满足稳定物理关系；异常时关系残差突然变大

**物理关系示例**:
- PWM / 电机速度 ↔ 姿态变化
- 气压计 ↔ 高度估计 ↔ 垂向速度
- GPS ↔ 位置与速度估计
- 电池电流 ↔ 推力需求 ↔ 控制输出

**技术实现**:
1. **邻接矩阵构建**: 利用 FP-Growth 算法对 RflyMAD 正常数据进行规则挖掘，识别强耦合传感器关系
2. **图神经网络**: 输入原始特征 Z [Batch, 41, 64] + 物理邻接矩阵 A [41, 41]
3. **特征校准**: GNN 根据物理规律推导"应该有的样子"，生成校准特征 Z'
4. **一致性残差**: 对比原始观测与校准预测，识别违反物理定律的异常

### 3.4 模块三: 可靠性感知的稀疏专家路由

**目标**: 根据飞行阶段、局部动态和模态不确定性动态选择异常专家，避免静态分配的泛化瓶颈。

**专家库设计**:

| 专家名称 | 处理异常类型 | 对应基线方法 | 核心能力 |
|---------|------------|------------|---------|
| **Burst Expert** | 尖峰、瞬态失稳、突变故障 | MAD-GAN | 识别瞬时跳变与脉冲干扰 |
| **Drift Expert** | 缓慢漂移、累计偏移、长期失配 | HMM | 捕捉微小的状态偏差 |
| **Spectral Expert** | 频谱纹理、振动模式和谐波异常 | GANomaly (4图像字段) | 识别频域结构畸变 |
| **Consistency Expert** | 单模态正常但联合不一致 | 物理图残差 | 判别违反物理拓扑约束的逻辑异常 |

**路由网络输入**:
- **飞行阶段向量**: [1, 64]，告知当前是"高动态起飞"还是"平稳悬停"
- **局部动态统计**: 滑动窗口内的方差（Variance）和峰度（Kurtosis）
- **模态完整性掩码**: [41]，传感器丢星时对应位设为 0
- **模态不确定性**: 41 个字段在编码过程中的重构熵（Entropy）

**路由机制**:
- **特征聚合层**: 拼接所有输入形成长向量（如维度 256）
- **专家路由层**: 矩阵乘法计算四个专家的原始得分
- **稀疏化处理**: Top-k 路由（通常 k=2），每帧只激活最相关的两个专家

### 3.5 模块四: 反事实模态扰动训练

**目标**: 使模型在模态缺失、时间错位、噪声污染和 sim-to-real 域偏移条件下仍具备稳定的开放集异常检测能力。

**五类扰动类型**:

| 扰动类型 | 实现手段 | 考察能力 |
|---------|---------|---------|
| **模态缺失** | 随机将某些 Token 置为全零，更新掩码 M | 路由回退能力：靠物理一致性维持判断 |
| **时间不同步** | 特定传感器字段 ±5 采样点随机平移 | 鲁棒性：识别总线延迟而非物理故障 |
| **局部噪声** | 注入高斯白噪声或脉冲噪声 | 去噪能力：通过 GNN 邻居信息"脑补"真实特征 |
| **频谱伪纹理** | 时频图中加入高频伪影或改变局部对比度 | 视觉抗扰性：防止过度依赖频谱图视觉美感 |
| **Sim-to-Real 扰动** | 特征向量域偏移缩放，模拟传感器漂移 | 泛化性：消除仿真环境过高信噪比带来的过拟合 |

**复合损失函数**:

1. **无关扰动不变性损失 L_inv**:  
   对单纯噪声、同步误差要求模型输出与干净样本一致  
   `|Score(Z_perturbed) - Score(Z_clean)| → 0`

2. **物理破坏敏感性损失 L_consist**:  
   人为制造违反物理规律的扰动（如 PWM 不变但调低加速度计），强制一致性专家报出极高残差

3. **模态缺失下的专家重组损失 L_route**:  
   掩码显示某模态缺失时，强制 Router 降权对应专家，让剩余专家接管

**输入输出维度**:
- 输入: 原始矩阵 Z [Batch, 41, 64]、扰动矩阵 Z_aug [Batch, 41, 64]、模态掩码 M [Batch, 41]
- 输出: 综合异常矩阵 [Batch, 41]、专家分配张量 [Batch, 4]、重构残差张量 [Batch, 41, 64]

## 六、核心创新点总结

1. **阶段感知多视图编码**: 首次将飞行阶段显式嵌入多模态异常检测框架，实现"特定阶段下的特定动态特征"精准建模
2. **物理一致性驱动的跨模态图学习**: 将无人机物理定律作为先验约束构建 GNN，识别单模态正常但物理关系被破坏的隐蔽异常
3. **可靠性感知的稀疏专家路由**: 根据信号动态、模态完整度和不确定性自适应激活专家，实现计算资源的按需分配
4. **反事实模态扰动训练体系**: 系统化模拟五类实际飞行干扰，通过复合损失函数实现开放集未知异常的鲁棒泛化
5. **统一的 Sim-to-Real 验证闭环**: 从 SIL → HIL → 真实飞行的全链路验证，确保方法的工程可实现性

## 八、关键术语表

| 术语 | 英文 | 解释 |
|------|------|------|
| 阶段感知 | Phase-Aware | 显式建模不同飞行阶段（起飞/悬停/巡航/降落）的动态特性差异 |
| 稀疏专家路由 | Sparse Expert Routing | 每帧只激活 Top-k 个最相关的专家子模型 |
| 物理一致性 | Physical Consistency | 传感器观测值之间满足物理定律（如 F=ma）的稳定关系 |
| 反事实扰动 | Counterfactual Perturbation | 人为制造违反物理规律或缺失模态的样本以增强鲁棒性 |
| Sim-to-Real | Simulation to Reality | 模型从仿真环境迁移到真实硬件平台的泛化能力 |
| 开放集检测 | Open-Set Detection | 识别训练阶段未见过的新型异常模式 |
| 模态完整性 | Modality Integrity | 各传感器模态在当前时刻的可信可用程度 |
